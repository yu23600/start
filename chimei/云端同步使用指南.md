# ☁️ 云端数据同步功能使用指南

## 📋 功能概述

Web应用现已支持真正的云端数据同步功能,可以将本地数据上传到云端服务器,或从云端下载其他用户分享的数据。

---

### 1. 确保服务已启动

**云端服务器** (端口 5000):


**Web应用** (端口 5001):
```powershell
cd D:\pythonProject3\web_app

```

### 2. 访问Web应用

在浏览器中打开: **http://localhost:5001**

## 🎯 功能说明

### ⬆️ 上传到云端

将本地的所有学校菜单数据上传到云端服务器。

**操作步骤:**
1. 在Web应用简洁模式页面底部找到"云端数据同步"区域
2. 点击 **"⬆️ 上传到云端"** 按钮
3. 等待上传完成,查看状态提示

**上传内容:**
- 所有学校的名称
- 每个学校的完整菜单
- 可用菜品列表(如果有)

**注意事项:**
- 上传会覆盖云端同名学校的旧数据
- 系统会自动记录上传者ID和时间戳
- 上传成功后会显示新增和更新的学校数量

---

### ⬇️ 从云端下载

从云端服务器下载最新的学校菜单数据到本地。

**操作步骤:**
1. 点击 **"⬇️ 从云端下载"** 按钮
2. 系统自动进行增量同步(只下载新数据)
3. 等待下载和合并完成

**智能合并策略:**
- **新增学校**: 直接添加到本地
- **已有学校**: 自动合并菜品,去重保留所有选项
- **冲突处理**: 保留本地和云端的所有菜品,不会丢失数据

**同步类型:**
- **全量同步**: 首次下载或手动触发,获取所有数据
- **增量同步**: 后续同步,只获取上次之后的更新

---

### 📊 查看云端统计

查看云端服务器的整体数据统计。

**显示信息:**
- 云端学校总数
- 总上传次数
- 热门学校排行榜(Top 10)
  - 点赞数
  - 下载次数

## 💡 使用场景

### 场景1: 分享你的学校菜单

1. 在本地编辑好学校菜单
2. 点击"保存当前菜单"
3. 点击"⬆️ 上传到云端"
4. 其他用户就可以下载你的数据了!

### 场景2: 获取其他学校的数据

1. 点击"⬇️ 从云端下载"
2. 系统自动合并所有可用数据
3. 在下拉列表中选择不同的学校查看

### 场景3: 多设备同步

**在学校电脑上:**
1. 编辑菜单 → 上传到云端

**在家里电脑上:**
1. 从云端下载 → 获得最新数据

## 🔧 技术细节

### API端点

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/upload` | POST | 上传学校数据 |
| `/api/download` | POST | 下载/同步数据 |
| `/api/stats` | GET | 获取统计数据 |
| `/api/schools` | GET | 获取学校列表 |
| `/api/menu/<school>` | GET/POST | 获取/保存菜单 |

### 数据存储

- **本地**: `web_app/cafeteria_data.json`
- **云端**: `app_server/cafeteria_cloud.db` (SQLite)

### 同步时间追踪

系统使用 `localStorage` 记录上次同步时间:
```javascript
localStorage.getItem('last_cloud_sync_time')
```

## ⚠️ 常见问题

### Q1: 上传失败怎么办?

**检查项:**
1. 云端服务器是否运行 (http://localhost:5000)
2. 网络连接是否正常
3. 查看浏览器控制台错误信息

**解决方法:**
- 重启云端服务器
- 检查防火墙设置
- 清除浏览器缓存后重试

### Q2: 下载后数据没有变化?

**可能原因:**
- 云端没有新数据
- 已是最新版本

**解决方法:**
- 点击"查看云端统计"确认有数据
- 检查状态提示信息

### Q3: 数据冲突如何处理?

**智能合并策略:**
- 菜品列表自动去重合并
- 保留所有 unique 的菜品
- 不会覆盖或删除任何数据

**示例:**
```
本地: 红烧肉, 鱼香肉丝
云端: 鱼香肉丝, 麻婆豆腐
合并后: 红烧肉, 鱼香肉丝, 麻婆豆腐
```

### Q4: 如何清空云端数据?

目前不支持直接删除,但你可以:
1. 停止云端服务器
2. 删除 `cafeteria_cloud.db` 文件
3. 重启服务器(会自动创建新数据库)
