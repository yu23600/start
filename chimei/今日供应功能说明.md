# 🍽️ 今日供应功能使用说明

## 问题场景
你是否遇到过这样的情况:
- 系统随机推荐了"红烧肉",但食堂今天根本没做这道菜
- 推荐的菜品虽然在你的菜单里,但今天刚好停供
- 每天都要面对"这个没有"、"那个卖完了"的尴尬

## ✨ 解决方案:今日供应管理

### 功能介绍
新增的**"设置今日供应"**功能让你可以标记每天实际供应的菜品,系统只会从真正有售的菜品中推荐!

### 使用步骤

#### 1️⃣ 点击"设置今日供应"按钮
在简洁模式的按钮区域,你会看到三个按钮:
- 保存当前菜单
- **设置今日供应** ← 点击这个
- 随机推荐一个菜品

#### 2️⃣ 勾选今日实际供应的菜品
弹出的窗口会显示你菜单中的所有菜品,你可以:
- ✅ **勾选** = 今天有这个菜
- ❌ **不勾选** = 今天没有/停供

**快捷操作:**
- **全选**: 所有菜品都供应
- **全不选**: 清空所有选择
- **反选**: 切换选中状态

#### 3️⃣ 保存设置
点击"保存设置"后:
- 系统会记住今天的供应情况
- 推荐时只会从勾选的菜品中选择
- 下次可以重新设置(比如明天换一批菜)

### 智能推荐效果

#### 简洁模式 - 随机推荐
```
✅ 之前: 可能推荐到没做的菜
✅ 现在: 只从今日供应的菜品中随机选择
```

#### 专业模式 - 智能推荐  
```
✅ 之前: 考虑营养需求,但可能推荐到没做的菜
✅ 现在: 先过滤出今日供应 → 再根据营养需求智能推荐
```

### 常见场景示例

**场景1: 周一食堂菜单**
- 总菜单: 红烧肉、鱼香肉丝、麻婆豆腐、西红柿炒蛋、清炒时蔬
- 今日供应: 红烧肉、鱼香肉丝、西红柿炒蛋 (麻婆豆腐和清炒时蔬今天不做)
- 操作: 只勾选这3个菜
- 结果: 推荐只会从这3个菜中选

**场景2: 临时调整**
- 早上设置了5个菜供应
- 中午发现某个菜卖完了
- 操作: 重新打开"设置今日供应",取消那个菜的勾选
- 结果: 下午的推荐不会再包含那个菜

**场景3: 全部供应**
- 如果某天所有菜都做
- 操作: 点击"全选"或者不设置(默认为全部供应)
- 结果: 推荐从所有菜品中选择

### 数据说明

#### 存储位置
今日供应数据保存在 `cafeteria_data.json` 文件中:
```json
{
  "我的学校": {
    "menu": "红烧肉\n鱼香肉丝\n...",
    "available_items": ["红烧肉", "鱼香肉丝", "西红柿炒蛋"],
    "last_modified": "2026-06-22 12:00"
  }
}
```

#### 默认行为
- **未设置时**: `available_items` 为空列表 `[]`,表示全部菜品都可用
- **已设置时**: 只从 `available_items` 中的菜品推荐
- **设为空时**: 如果故意设置为空,系统会提示"暂无供应菜品"

### 温馨提示

💡 **每天早上花1分钟设置今日供应**
- 看一眼食堂今天的菜单
- 勾选实际供应的菜品
- 一整天推荐都不会出错

💡 **灵活调整**
- 可以随时修改今日供应
- 不同学校可以有不同的供应设置
- 支持快速全选/全不选/反选

💡 **与智能推荐完美结合**
- 今日供应是第一层过滤
- 营养规则是第二层过滤
- 最终推荐 = 今日供应 ∩ 营养匹配

### 技术细节

**数据结构:**
- 每个学校独立维护自己的今日供应列表
- 不影响原有的菜单数据
- 向后兼容(旧数据自动适配)

**推荐逻辑:**
```
1. 获取菜单所有菜品
2. 如果有今日供应设置 → 过滤出供应菜品
3. [专业模式] 根据身体状况进一步筛选
4. 从最终候选列表中推荐
```

**错误处理:**
- 今日供应菜品不在菜单中 → 提示检查设置
- 今日供应为空 → 确认是否真的没有菜品
- 所有菜品都被排除 → 给出详细原因和建议
