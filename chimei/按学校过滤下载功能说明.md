# 🎯 按学校过滤下载功能说明

## 📋 功能概述

现在用户从云端下载数据时,**只会获取当前学校的食谱**,不会下载其他学校的数据。

---

## 🔧 技术实现

### 1. 后端API修改

**文件**: `app_server.py`

**新增参数**: `school_name` (可选)

```python
@app.route('/api/download', methods=['POST'])
def download_data():
    """
    请求体格式:
    {
        "last_sync_time": "上次同步时间 (可选)",
        "school_name": "学校名称 (可选，如果提供则只返回该学校的数据)"
    }
    """
    school_name = data.get('school_name', None)
    
    if school_name:
        # 只查询指定学校的数据
        cursor.execute('''
            SELECT * FROM school_menus 
            WHERE school_name = ? AND updated_at > ?
            ORDER BY updated_at DESC
        ''', (school_name, last_sync_time))
    else:
        # 查询所有学校(兼容旧版本)
        ...
```

### 2. 前端逻辑修改

**文件**: `web_app/static/script.js`

**修改点**: `downloadFromCloud()` 函数

```javascript
// 传递当前学校名称
```

## 🎯 使用场景

### 场景1: 北京大学用户下载

**操作流程:**
1. 用户选择学校: **北京大学**
2. 点击"⬇️ 从云端下载"
3. 系统请求: `{ school_name: "北京大学" }`
4. 云端返回: **只有北京大学的食谱**
5. 本地更新: **仅更新北京大学的数据**

**结果:**
- ✅ 获得北京大学的最新食谱
- ❌ 不会获得清华大学、复旦大学等其他学校的数据

### 场景2: 切换学校后下载

**操作流程:**
1. 用户原本在: **北京大学**
2. 点击"✏️ 修改学校" → 选择 **清华大学**
3. 点击"⬇️ 从云端下载"
4. 系统请求: `{ school_name: "清华大学" }`
5. 云端返回: **只有清华大学的食谱**

**结果:**
- ✅ 获得清华大学的最新食谱
- ❌ 不会获得北京大学的数据

## 💡 优势

### 1. 数据精准
- 只下载需要的学校数据
- 避免无关数据干扰
- 减少网络流量

### 2. 隐私保护
- 不同学校数据隔离
- 用户只看到自己学校的内容
- 不会泄露其他学校信息

### 3. 性能优化
- 数据传输量更小
- 下载速度更快
- 本地存储更简洁

### 4. 用户体验
- 界面更清爽(只有一个学校)
- 操作更符合直觉
- 不会混淆不同学校的数据

## 🔄 兼容性

### 向后兼容

如果某些客户端没有传递 `school_name` 参数:

```javascript
// 旧版本客户端
body: JSON.stringify({
    last_sync_time: lastSyncTime || null
    // 没有 school_name
})
```

**服务器行为:**
- 检测到 `school_name` 为 `null`
- 返回**所有学校**的数据(保持原有行为)
- 确保旧版本客户端正常工作

### 新版本特性

```javascript
// 新版本客户端
body: JSON.stringify({
    last_sync_time: lastSyncTime || null,
    school_name: currentSchool  // 传递学校名称
})
```

**服务器行为:**
- 检测到 `school_name` 有值
- 只返回**指定学校**的数据
- 响应中包含标记: `"filtered_by_school": true`
